# Prompt工程知识库

## 一、论文生成Prompt模式

### 模式1：模板化生成（Template-based）
**来源：** CAJAL
```
系统：你是[学科]领域的资深研究者。请按照以下结构撰写论文的[章节]部分：
- 要求：[具体要求]
- 字数：[X-Y]字
- 引用：必须引用真实文献
- 语言：[中文/英文]
```

### 模式2：角色扮演生成（Role-play）
**来源：** CrewAI, HeShen-1
```
Agent角色：你是一位在[领域]有10年研究经验的教授。
目标：撰写高质量的[章节]，确保逻辑严密、引用充分。
背景：你发表过[X]篇SCI论文，精通[方法]。
```

### 模式3：迭代细化（Iterative Refinement）
**来源：** NORA, HalfSeed
```
第1轮：生成初稿
第2轮：根据评审意见修改
第3轮：再次评审，直到评分>=阈值
```

### 模式4：对抗生成（Adversarial）
**来源：** NORA, HalfSeed
```
Writer Prompt：基于证据撰写[章节]
Reviewer Prompt：从[维度1-5]严格评审，给出具体修改建议
循环：Writer根据Review修改，直到通过
```

## 二、各章节Prompt模板

### Introduction生成Prompt
```
基于以下文献综述，撰写论文Introduction部分：

文献摘要：
{literature_summary}

要求：
1. 从宏观背景引入（2-3句）
2. 综述现有研究（3-5段）
3. 指出研究空白（1段）
4. 阐述本研究目的和创新点（1段）
5. 引用文献使用[Author, Year]格式
6. 总字数800-1200字
```

### Methods生成Prompt
```
基于以下实验设计和数据，撰写Materials & Methods部分：

实验信息：
- 研究区域：{study_area}
- 采样方案：{sampling_plan}
- 分析方法：{analysis_methods}
- 仪器设备：{instruments}

要求：
1. 按"采样→前处理→分析→统计"顺序
2. 方法描述需足够可重复
3. 统计方法需说明软件和参数
4. 引用国标/行标方法
```

### Results生成Prompt
```
基于以下数据统计结果，撰写Results部分：

数据摘要：
{statistical_results}

图表信息：
{figure_table_info}

要求：
1. 只描述数据，不解释原因
2. 图表自明，文字补充关键信息
3. 数据与图表不重复
4. 显著性标注完整
```

### Discussion生成Prompt
```
基于以下Results和文献对比，撰写Discussion部分：

本研究结果：
{results_summary}

文献对比：
{literature_comparison}

要求：
1. 先总结主要发现（1-2句）
2. 与文献对比（同异分析）
3. 机制解释（为什么）
4. 本研究意义
5. 局限性
6. 展望
```

## 三、文献搜索Prompt

### arXiv搜索Prompt
```
为以下研究主题搜索最相关的论文：

研究主题：{topic}
搜索关键词：{keywords}
时间范围：{year_range}

请返回：
1. 最相关的10篇论文
2. 每篇的：标题、作者、摘要、arXiv ID
3. 按相关性排序
```

### 文献综述Prompt
```
基于以下搜索结果，撰写文献综述：

论文列表：
{paper_list}

要求：
1. 按主题分类组织（不要按时间线）
2. 突出各研究的异同
3. 指出研究趋势
4. 识别研究空白
5. 每段至少引用3篇文献
```

## 四、评审Prompt

### 单维度评审
```
请从[维度]角度评审以下论文段落：

段落内容：{content}

评分标准（1-10分）：
- 1-3分：严重问题
- 4-6分：需要改进
- 7-8分：良好
- 9-10分：优秀

请给出：
1. 评分
2. 具体问题
3. 修改建议
```

### 综合评审（NORA模式）
```
请对以下论文进行全面评审：

论文内容：{paper_content}

评审维度及权重：
1. Novelty（30%）- 创新性，门槛>=6.5
2. Rigor（25%）- 严谨性，门槛>=7.0
3. Literature（20%）- 文献覆盖，门槛>=6.5
4. Clarity（15%）- 清晰度，门槛>=6.0
5. Impact（10%）- 影响力，门槛>=6.0

输出格式：
- 每维度评分+理由
- 加权总分
- 是否通过（均分>=7.5且所有门槛达标）
- 具体修改建议（按优先级排序）
```
