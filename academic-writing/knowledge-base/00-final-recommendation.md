# 科研AI生态系统 - 最终方案

## 核心结论

经过分析20+个GitHub项目，没有单一项目能满足中文科研论文自动生成的完整需求。最佳方案是**混合架构**：提取各项目精华，组合成适合环境科学研究的专用系统。

---

## 推荐架构：NORA + CrewAI + LangGraph 混合

### 为什么这个组合

| 需求 | 最佳方案 | 来源项目 |
|------|---------|---------|
| 工作流定义 | Skills-first (Markdown) | NORA |
| Agent协作 | 角色型 + 层级委派 | CrewAI |
| 状态管理 | 图执行 + 检查点 | LangGraph |
| 论文评审 | 对抗式 + 评分系统 | NORA |
| LaTeX输出 | 模板化 + PI-LOCK | HalfSeed + NORA |
| 中文支持 | DeepSeek + 中文Prompt | HeShen-1 |
| 文献RAG | Agentic RAG + 元数据丰富 | PaperQA2 |
| 本地运行 | Ollama + 无API依赖 | CAJAL |

## 5个核心Agent

```
┌──────────────────────────────────────────────┐
│              Orchestrator（编排器）            │
│         管理流程、传递上下文、控制节奏          │
└──────────────┬───────────────────────────────┘
               │
    ┌──────────┼──────────┬──────────┐
    ▼          ▼          ▼          ▼
┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
│Literature│ │Analyst │ │Writer  │ │Reviewer│
│ Scout   │ │        │ │        │ │        │
│文献侦察  │ │数据分析 │ │论文撰写 │ │同行评审 │
└────────┘ └────────┘ └────────┘ └────────┘
               │
               ▼
         ┌────────┐
         │Formatter│
         │格式化   │
         └────────┘
```

### Agent职责与工具

**Literature Scout**
- arXiv / Semantic Scholar / CNKI搜索
- PDF解析与结构化
- 文献综述生成
- 引用管理

**Analyst**
- Python统计分析（PCA/HCA/相关性/回归）
- 图表生成（matplotlib/seaborn）
- 数据→文字转换

**Writer**
- 逐章生成论文
- 句式库调用
- 引用插入
- LaTeX/Markdown输出

**Reviewer**
- 5维度评分（科学性30%/创新性25%/文献20%/逻辑15%/可读10%）
- 具体修改建议
- 硬门槛检查

**Formatter**
- 期刊模板选择
- LaTeX编译
- 参考文献格式化
- PDF生成

## 评审系统设计（借鉴NORA）

### 评分维度（环境科学版）

| 维度 | 权重 | 硬门槛 | 说明 |
|------|------|--------|------|
| 科学性 | 30% | >=7.0 | 方法正确、数据可靠 |
| 创新性 | 25% | >=6.5 | 新发现、新方法、新视角 |
| 文献支撑 | 20% | >=6.5 | 引用充分、对比充分 |
| 逻辑性 | 15% | >=6.0 | 论证链完整、因果清晰 |
| 可读性 | 10% | >=6.0 | 表达清晰、结构合理 |

**接受条件：** 加权均分 >= 7.5 且所有维度 >= 硬门槛

### 评审流程
```
Writer生成初稿 → Reviewer评分 → 通过?
├── 是 → 进入格式化
└── 否 → 返回修改建议 → Writer修改 → 再次评审
    └── 最多4轮，第4轮降低门槛至6.5
```

## 记忆系统（三层架构）

```
长期记忆         中期记忆          短期记忆
(持久化)         (会话级)          (上下文)
━━━━━━━━━       ━━━━━━━━         ━━━━━━━━
知识库           handoff.json      Agent对话
MEMORY.md        project-state     Task输出
Git历史          TELEMETRY         当前状态
文献数据库
```

### 关键机制
- **PI-LOCK：** 保护用户手写内容不被覆盖
- **Git版本控制：** 每次编辑自动commit
- **handoff.json：** 跨会话状态传递
- **知识库：** 领域知识 + 写作规则 + 句式库
